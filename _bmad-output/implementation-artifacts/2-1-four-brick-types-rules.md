# Story 2.1: 브릭 4종 규칙 구현

Status: review

## Story

As a 플레이어,  
I want 일반, 강체, 폭발, 아이템 브릭이 서로 다른 규칙으로 반응하기를,  
so that 같은 공과 패들 루프 안에서도 스테이지를 읽고 판단하는 재미가 생긴다.

## Acceptance Criteria

1. Stage 1/2에서 일반, 강체, 폭발, 아이템 브릭 4종이 시각적으로 구분되어 생성될 수 있어야 한다.
2. 일반 브릭은 1회 충돌로 즉시 파괴되어야 한다.
3. 강체 브릭은 3 내구도를 가지며, 피격 때마다 내구도 상태가 시각적으로 드러나야 한다.
4. 폭발 브릭은 파괴될 때 인접 브릭에 추가 피해를 주거나 즉시 파괴 효과를 전달해야 한다.
5. 아이템 브릭은 파괴될 때 이후 스토리에서 사용할 드롭 이벤트 훅을 남겨야 한다.
6. 브릭 타입 규칙은 시스템과 순수 함수로 분리되어 테스트 가능해야 한다.

## Dev Notes

- 이번 단계의 핵심은 타입별 차이를 읽히게 만드는 것이지, 모든 후속 시스템을 완성하는 것이 아니다.
- 아이템 브릭은 실제 드롭 오브젝트 생성 전이라도 `drop-requested` 이벤트 수준의 훅을 반드시 남긴다.
- 폭발 브릭의 영향 범위는 과도한 카오스를 피하기 위해 인접 1칸 수준으로 시작한다.
