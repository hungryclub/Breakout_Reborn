# Breakout_Reborn 브레인스토밍 요약

## 출처

- 기반 문서: `docs/init/Breakout_Reborn_Plan.md`
- 진행 방식: BMAD Party Mode 스타일의 다각도 브레인스토밍

## 핵심 결론

`Breakout_Reborn`는 초기에 콘텐츠 양으로 승부하는 게임이 아니라, 아래 세 가지로 인상을 남겨야 합니다.

1. 한 손으로도 즉시 이해되고 반응하는 모바일 조작감
2. 시작 직후부터 강하게 느껴지는 타격감과 콤보 연출
3. 짧은 플레이를 반복하게 만드는 가벼운 로그라이트식 변주

## 추천 포지셔닝

- "DX-Ball 감성을 한 손 모바일 플레이에 맞게 되살린 게임"
- "짧게 즐기기 좋은 세로형 브릭브레이커"
- "연출과 손맛으로 차별화되는 스킬 액션형 벽돌깨기"

## MVP 방향

### 유지할 요소

- 상대 드래그 기반 패들 조작
- 탭으로 발사 및 액티브 아이템 사용
- 핵심 벽돌 4종: 일반, 강철, 폭발, 아이템 벽돌
- 소규모 파워업 풀: 패들 확장, 파이어볼, 멀티볼, 자석
- 스테이지 진행에 따라 조금씩 상승하는 속도와 난이도
- 히트 스톱, 파티클, 화면 흔들림을 활용한 콤보 피드백
- 스테이지 사이에 제공되는 간단한 런 기반 성장 또는 보상

### 당장은 제외할 요소

- 복잡한 스킬트리
- 여러 개의 게임 모드
- 비중이 큰 서사 연출
- 대규모 코스메틱 및 메타 시스템
- 본격적인 과금 설계

## 첫 3분 설계 원칙

- 게임 시작 후 몇 초 안에 첫 발사가 가능해야 함
- 첫 플레이 안에서 만족스러운 콤보 경험이 나와야 함
- 30초 이내에 첫 보상이나 해금 요소를 보여줘야 함
- 실패 직후 즉시 재도전이 가능해야 함
- HUD는 스테이지, 점수, 콤보, 목숨 정도로 최소화해야 함

## UX 원칙

- 정보량은 적게, 피드백은 크게
- 손가락이 공의 궤적을 가리지 않도록 설계
- 장식보다 공의 움직임과 충돌 가독성을 우선
- 시스템 복잡도보다 타격 연출이 더 크게 느껴지게 구성

## 기술 우선순위

- 터치 입력 지연이 없는지 가장 먼저 검증
- 모바일 브라우저에서 60fps를 안정적으로 유지
- 브라우저별 충돌 판정 차이를 조기에 확인
- 9:16 기준 반응형 캔버스 레이아웃을 안정적으로 구성
- 짧은 세션 기준의 일시정지, 복구, 재진입 처리 검토

## 프로토타입 우선순위

1. 패들 입력과 발사 감각
2. 공 반사 물리와 패들 위치별 반사각 제어
3. 벽돌 충돌 및 파괴 피드백
4. 반응형 캔버스와 HUD 가독성
5. 최소 수준의 레벨 생성과 파워업 드롭 루프

## 최종 제안

가장 강한 버전의 `Breakout_Reborn`는 아래와 같습니다.

"즉시 손에 붙는 세로형 모바일 브릭브레이커로, 짧은 재도전과 매번 조금씩 달라지는 플레이를 통해 반복 욕구를 만드는 게임"

향후 확장은 아래 순서를 권장합니다.

1. 벽돌 상호작용 다양화
2. 런 변주 요소 강화
3. 보스 스테이지 또는 특수 이벤트 추가
4. 메타 성장 시스템 확장
