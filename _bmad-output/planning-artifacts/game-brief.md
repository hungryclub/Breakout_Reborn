# Game Brief: Breakout_Reborn

**Date:** 2026-03-28  
**Author:** Dhlee  
**Status:** GDD 작성을 위한 초안 완료

---

## Executive Summary

`Breakout_Reborn`는 고전 브릭브레이커의 손맛을 현대 모바일 세로 화면 UX에 맞게 재해석한 하이브리드 캐주얼 게임이다. 핵심 목표는 한 손으로 즉시 이해되는 조작, 강한 타격감, 2~4분 단위의 짧은 세션 반복을 만드는 재도전 템포를 결합하는 것이다. 초기 버전은 콘텐츠 양보다 입력 반응성, 충돌 피드백, 간단한 런 기반 변주, 빠른 재시도 흐름에 집중한다. 동시에 이 프로젝트는 단순 프로토타입이 아니라, “작지만 상용 게임처럼 느껴지는 완성도”를 제품 기준으로 삼는다.

---

## Game Vision

### Core Concept

고전 `DX-Ball`류의 벽돌깨기를 세로형 모바일 플레이에 맞춰 재설계한 액션 게임이다. 상대 드래그 기반 패들 조작과 탭 기반 발사/액티브 사용을 통해 손가락이 화면을 가리지 않으면서도 직관적인 플레이를 제공한다.

### Elevator Pitch

“`Breakout_Reborn`는 DX-Ball 감성의 타격감을 한 손 모바일 플레이에 맞게 되살린 세로형 브릭브레이커로, 짧고 강한 재도전 루프와 가벼운 로그라이트식 변주를 결합한 게임이다.”

### Vision Statement

누구나 몇 초 안에 시작할 수 있지만, 반복할수록 손맛과 연쇄 파괴, 짧은 성장 선택의 재미에 빠져드는 현대적 브릭브레이커를 만든다.

### Commercial Quality Statement

`Breakout_Reborn`의 차별화는 콘텐츠 양이 아니라 체감 품질에서 나와야 한다. 따라서 모든 시스템과 화면은 “즉시 반응한다”, “명확하게 읽힌다”, “강하게 보상한다”, “다시 누르게 만든다”라는 네 가지 상용 품질 기준 아래 설계한다.

## Target Market

### Primary Audience

- 모바일에서 짧은 세션 게임을 즐기는 캐주얼 액션 유저
- 브릭브레이커, 아케이드, 로그라이트 변주를 좋아하는 20~40대 유저
- 과한 학습 없이 즉시 플레이 가능한 게임을 선호하는 사용자

### Secondary Audience

- DX-Ball, Arkanoid류 고전 게임 향수를 가진 복귀 유저
- 모바일 웹에서 설치 없이 가볍게 즐길 수 있는 게임을 찾는 사용자

### Market Context

모바일 캐주얼 시장에서는 빠른 진입성과 짧은 세션 유지력이 중요하다. `Breakout_Reborn`는 익숙한 장르 규칙을 활용하면서도 세로형 UI, 강한 피드백, 즉시 재도전 구조, 런마다 약간 달라지는 선택지를 통해 차별화한다. 핵심 리스크는 “그냥 DX-Ball 같은 게임”으로 보이는 것이며, 이를 피하기 위해 모바일 특화 UX와 짧은 세션 리텐션 장치를 전면에 둔다.

## Game Fundamentals

### Core Gameplay Pillars

1. 즉시 반응하는 한 손 조작
2. 과장된 타격감과 연쇄 파괴 쾌감
3. 짧은 세션 기반의 반복 욕구
4. 복잡하지 않지만 매번 조금씩 달라지는 런 변주

### Primary Mechanics

- 상대 드래그로 패들 이동
- 탭으로 공 발사 및 일부 액티브 아이템 사용
- 벽돌 충돌 및 반사각 제어
- 콤보 기반 점수 상승과 시각 효과 강화
- 벽돌 파괴 시 파워업 드롭
- 스테이지 클리어 후 난이도 점진 상승

### Player Experience Goals

- 게임 시작 즉시 “손에 붙는다”는 감각을 줄 것
- 첫 1분 안에 강한 타격감과 콤보 연출을 체험시킬 것
- 실패해도 즉시 다시 도전하고 싶게 만들 것
- 첫 3분 안에 보상, 선택, 재도전 욕구가 한 번씩 모두 발생하게 할 것
- 콘텐츠 양이 적더라도 “싸 보이지 않는다”는 인상을 줄 것

### Quality Pillars

1. 입력은 즉시 반응해야 하며, 지연보다 신뢰가 먼저 느껴져야 한다.
2. 충돌, 콤보, 클리어, 실패는 모두 짧고 강한 시청각 피드백을 가져야 한다.
3. 플레이 중 시선은 언제나 메인 공, 충돌 지점, 다음 판단 지점으로 자연스럽게 유도되어야 한다.
4. 실패와 재도전은 끊김이 아니라 템포를 만드는 경험이어야 한다.
5. 콘텐츠 수보다 상호작용 밀도와 순간 품질을 더 중요한 성공 기준으로 삼는다.

## Scope and Constraints

### Target Platforms

- 모바일 웹 브라우저 우선
- 세로 화면 9:16 중심
- 향후 패키징 가능성을 열어두되 초기 목표는 웹 플레이

### Development Timeline

- 1차 목표: 조작감과 타격감 검증용 프로토타입
- 2차 목표: MVP 완성
- 3차 목표: GDD 기반 콘텐츠 확장 검토

### Budget Considerations

- 초기 단계에서는 저비용 검증 중심
- 과한 아트 자산과 대규모 라이브 운영 요소는 후순위

### Team Resources

- 소규모 또는 솔로 개발 전제
- BMAD 기반 기획/설계/개발 보조 워크플로우 활용

### Technical Constraints

- 모바일 브라우저 환경에서 안정적인 60fps 유지 필요
- 다양한 화면비에서 반응형 레이아웃 보장 필요
- 브라우저별 터치 입력 및 충돌 일관성 검증 필요
- 조작 감도, 충돌 규칙, 재시도 흐름은 감성적 표현이 아니라 명시 가능한 규칙으로 내려와야 함
- 시청각 연출 품질은 “가능하면”이 아니라 제품 정체성을 구성하는 필수 항목으로 취급해야 함
- 저사양 환경에서도 핵심 조작감과 판독성은 유지되어야 하며, 연출은 품질 단계별로 축소 가능해야 함

## Reference Framework

### Inspiration Games

- `DX-Ball`
- `Arkanoid`
- 현대 모바일 하이브리드 캐주얼 아케이드 게임 전반

### Competitive Analysis

전통 브릭브레이커 계열은 규칙 이해는 쉽지만, 모바일 환경에서는 조작 스트레스와 약한 피드백으로 이탈이 발생하기 쉽다. `Breakout_Reborn`는 한 손 조작 최적화와 과감한 충돌 피드백으로 그 간극을 메운다.

### Key Differentiators

- 손가락이 패들을 가리지 않는 상대 드래그 조작
- 세로형 화면에 맞춘 가독성 높은 플레이 필드
- 히트 스톱, 파티클, 화면 흔들림 중심의 쥬시한 연출
- 짧은 런 안에서 느껴지는 가벼운 로그라이트식 변화
- 실패 직후 다시 붙잡는 빠른 재도전 UX
- 작은 볼륨으로도 상용 아케이드처럼 느껴지는 일관된 폴리시

## Content Framework

### World and Setting

초기 MVP에서는 서사보다 시스템과 플레이 감각이 우선이다. 세계관은 최소화하고, 시각 테마는 네온/아케이드 감성 또는 현대적 미니멀 아케이드 톤 중 하나로 정리한다.

### Narrative Approach

내러티브는 중심 요소가 아니다. 게임의 이야기는 플레이 중 발생하는 콤보, 위기, 역전, 보상 흐름으로 전달한다.

### Content Volume

- 벽돌 타입 4종 중심
- 파워업 소수 정예 구성
- 단일 메인 모드 우선
- 스테이지 수는 적더라도 반복 플레이가 가능하도록 설계
- 최소 메타 진행은 1개 축으로 제한해 과도한 시스템 확장을 막음

## Art and Audio Direction

### Visual Style

읽기 쉬운 배경 위에 선명한 벽돌 컬러와 명확한 충돌 이펙트를 배치한다. 작은 디테일보다 파괴 순간의 큰 반응이 중요하다.

### Audio Style

짧고 명확한 충돌 효과음, 콤보가 올라갈수록 강화되는 보상감 있는 사운드, 재도전을 방해하지 않는 빠른 루프형 배경음이 적합하다.

### Production Approach

초기에는 단순한 도형 기반과 최소한의 효과 자산으로 프로토타입을 만든다. 이후 검증된 손맛을 기준으로 시각 연출과 오디오를 강화한다.

### Commercial Presentation Principles

- 모든 핵심 이벤트는 시각, 사운드, 화면 반응이 한 세트로 느껴져야 한다.
- 버튼, 보상 카드, 실패 화면, 스테이지 전환은 같은 리듬과 모션 언어를 공유해야 한다.
- 강한 연출은 많을수록 좋은 것이 아니라, 중요한 순간에 정확히 집중되어야 한다.
- 저예산 제작이어도 싸구려처럼 느껴지는 과도한 혼잡, 약한 피드백, 느린 전환은 허용하지 않는다.

## Risk Assessment

### Key Risks

- 조작감이 기대만큼 즉각적이지 않을 가능성
- 장르 특성상 초반 반복성이 빠르게 지루해질 가능성
- 모바일 웹 환경에서 성능과 입력 안정성이 흔들릴 가능성
- 보상 구조가 약하면 재방문 동기가 부족할 가능성

### Technical Challenges

- 터치 입력 지연 최소화
- 반사 물리의 일관성 확보
- 브라우저별 렌더링/충돌 오차 대응
- 세션 복구와 재진입 처리

### Market Risks

- 브릭브레이커 장르의 익숙함이 신선함 부족으로 해석될 수 있음
- 콘텐츠 볼륨이 적으면 초기 리텐션이 약할 수 있음
- 로그라이트 요소가 약하면 차별화가 흐려지고, 반대로 과하면 MVP 범위를 망칠 수 있음

### Mitigation Strategies

- 가장 먼저 조작감과 충돌 피드백을 프로토타입으로 검증
- 첫 3분 보상 구조와 즉시 재도전 루프를 강하게 설계
- 콘텐츠 확장보다 핵심 파워업/벽돌 상호작용의 밀도를 높임
- 메타 진행은 “한 가지 분명한 축”만 두고, 나머지는 후속 단계로 넘김
- 폴리시 레이어와 품질 기준을 초기에 문서화해, 구현 후반의 임시방편 연출 누적을 방지

## Success Criteria

### MVP Definition

아래를 만족하면 MVP로 본다.

- 한 손 조작이 자연스럽고 입력 지연이 거의 없음
- 기본 브릭브레이커 루프가 안정적으로 동작함
- 벽돌 4종과 핵심 파워업이 구현됨
- 콤보/이펙트 기반 손맛이 충분히 전달됨
- 짧은 스테이지 반복과 빠른 재도전이 가능함
- 스테이지 사이 최소 1개의 선택형 보상 또는 성장 흐름이 작동함
- 작은 콘텐츠 규모에도 첫인상과 재도전 경험이 상용 게임 수준으로 정돈되어 있음

### Success Metrics

- 첫 플레이에서 10초 안에 조작을 이해할 수 있을 것
- 첫 3분 안에 만족스러운 연쇄 파괴 경험과 첫 보상이 발생할 것
- 실패 후 5초 안에 재도전할 수 있을 것
- 내부 플레이테스트 기준 “한 판 더” 반응이 반복적으로 관찰될 것
- 멀티볼, 폭발 벽돌, 보상 선택 상황에서도 무엇을 봐야 하는지 혼란이 적을 것
- 첫 10분 플레이 후 “저예산 프로토타입”보다 “작지만 잘 다듬어진 게임”에 가깝다는 정성 피드백을 확보할 것

### Launch Goals

- GDD 작성에 필요한 방향성을 고정
- 핵심 프로토타입 우선순위를 명확히 함
- 확장 전에 무엇을 의도적으로 제외할지 합의함
- 향후 D1/재방문 검증에 쓸 기준 지표 초안을 확보

### Release Quality Bar

- 첫 조작에서 지연이나 억울함이 느껴지지 않을 것
- 충돌, 파괴, 콤보, 실패, 재도전의 피드백 톤이 일관될 것
- 중요한 순간에 시선 유도와 HUD 정보 위계가 무너지지 않을 것
- 재도전 흐름이 한 템포 안에 이어져 세션 이탈보다 재개를 먼저 유도할 것
- 적은 콘텐츠량이 약점으로 읽히지 않도록 상호작용 밀도와 연출 완성도로 보완할 것

### A. Research Summary

- 브레인스토밍 결과상 이 게임의 핵심 경쟁력은 콘텐츠 양이 아니라 첫 3분의 손맛이다.
- UX 측면에서 “한 손으로 읽히고, 즉시 때리고, 크게 쾌감”이 핵심 원칙으로 도출되었다.
- 기술 측면에서 입력 지연, 60fps 유지, 반응형 캔버스 안정성이 선행 검증 항목이다.
- 추가 토론 결과, 세션 길이, 실패 직후 재시도 UX, 최소 메타 진행, 성공 지표 명시가 브리프 보강 포인트로 도출되었다.

### B. Stakeholder Input

- 현재 입력 자료는 초기 기획서와 내부 브레인스토밍 요약 기준으로 정리되었다.
- 추가 이해관계자 의견은 아직 반영되지 않았다.
