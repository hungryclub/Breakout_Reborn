# Breakout_Reborn - Game Design Document

**Author:** Dhlee  
**Game Type:** 하이브리드 캐주얼 브릭브레이커 / 라이트 로그라이트  
**Target Platform(s):** 모바일 웹 브라우저, 세로형 9:16

---

## Executive Summary

### Core Concept

`Breakout_Reborn`는 고전 `DX-Ball` 계열 브릭브레이커를 모바일 세로 화면과 한 손 조작에 맞게 재해석한 액션 게임이다. 플레이어는 화면 하단에서 상대 드래그로 패들을 조작하고, 공의 반사각을 제어하며 벽돌을 파괴한다. 핵심 목표는 “즉시 손에 붙는 조작감”, “강한 타격감”, “짧고 반복 가능한 런 구조”를 하나의 루프로 엮는 것이다.

### Target Audience

- 짧은 세션의 모바일 캐주얼 액션 게임을 선호하는 사용자
- 아케이드 손맛과 재도전 구조를 좋아하는 사용자
- 고전 브릭브레이커 향수를 현대 UX로 다시 즐기고 싶은 사용자

### Unique Selling Points (USPs)

- 손가락이 패들을 가리지 않는 상대 드래그 조작
- 히트 스톱, 파티클, 화면 흔들림 기반의 강한 충돌 피드백
- 2~4분 세션 안에서 완결되는 빠른 재도전 구조
- 브릭브레이커에 가벼운 로그라이트식 보상 선택을 얹은 변주
- 적은 콘텐츠량으로도 상용 아케이드처럼 느껴지는 일관된 폴리시

## Goals and Context

### Project Goals

1. 모바일 웹에서 즉시 플레이 가능한 브릭브레이커를 만든다.
2. 첫 3분 안에 손맛, 보상, 재도전 욕구를 모두 전달한다.
3. 적은 콘텐츠량으로도 반복 플레이가 가능한 시스템 밀도를 확보한다.
4. 작은 볼륨이어도 “프로토타입”보다 “잘 다듬어진 상용 게임”에 가깝게 느껴지도록 만든다.

### Background and Rationale

브릭브레이커 장르는 규칙 이해가 쉽고 즉각적인 만족감을 주기 좋지만, 모바일 환경에서는 손가림, 약한 피드백, 반복감 때문에 쉽게 이탈할 수 있다. `Breakout_Reborn`는 조작 스트레스를 줄이고, 충돌 연출을 키우고, 짧은 세션 리듬과 라이트한 성장 선택을 더해 현대적인 플레이 감각으로 재구성한다.

### Commercial Quality Requirements

- 입력, 충돌, 시선 유도, 재도전 템포는 게임성뿐 아니라 제품 품질의 핵심 요구사항이다.
- 모든 주요 순간은 시각, 사운드, 화면 반응이 한 세트로 작동해야 한다.
- 연출은 화려함보다 우선순위가 중요하며, 메인 공과 핵심 판단 지점을 가리지 않아야 한다.
- 저사양 환경에서는 연출 밀도를 줄일 수 있어도 조작감과 판독성은 절대 훼손하지 않는다.

## Core Gameplay

### Game Pillars

1. 조작은 단순하지만 공의 각도 제어에는 숙련이 개입한다.
2. 벽돌 하나를 깨는 순간도 시각적 보상이 크게 느껴져야 한다.
3. 실패는 짧고, 재도전은 빠르고, 매 판은 약간씩 달라야 한다.

### Core Gameplay Loop

1. 플레이어가 스테이지에 진입한다.
2. 탭으로 공을 발사한다.
3. 상대 드래그로 패들을 이동해 반사각을 제어한다.
4. 벽돌을 파괴하며 점수, 콤보, 파워업 드롭을 획득한다.
5. 스테이지를 클리어하면 보상 선택 또는 간단한 성장 선택을 한다.
6. 다음 스테이지에서 더 빠른 공 속도와 조합된 벽돌 배치를 상대한다.
7. 실패하면 즉시 재도전하거나 런을 종료한다.

### Win/Loss Conditions

- 승리: 해당 스테이지의 파괴 가능한 모든 벽돌 제거
- 런 승리: 정해진 스테이지 묶음 완료
- 패배: 모든 생명을 소진하거나 현재 보유 중인 모든 공이 추락한 뒤 생명도 남지 않은 상태

## Game Mechanics

### Primary Mechanics

#### 패들 이동

- 입력 방식: 화면 하단 아무 위치나 터치 후 좌우 드래그
- 조작 방식: 절대 위치 이동이 아니라 상대 이동
- 목적: 손가락이 패들을 직접 가리지 않게 하고 공 궤적 가독성을 보장

#### 공 반사

- 기본 규칙: 입사각 기반 반사
- 패들 각도 보정: 패들 중앙은 수직에 가까운 각도, 가장자리는 더 큰 사선 각도 부여
- 반사각 제한: 진행 방향 기준 최소 20도, 최대 75도 범위로 제한한다.
- 패들 중앙 20% 구간은 안정 반사 구역, 양끝 20% 구간은 강한 각도 변화 구역으로 취급한다.
- 동일 프레임 중복 충돌로 공이 비정상 반전되지 않도록 1회 충돌 후 짧은 재충돌 방지 윈도우를 둔다.

#### 벽돌 파괴

- 충돌 시 벽돌 체력 감소
- 체력이 0이 되면 파괴 및 이펙트 재생
- 일부 벽돌은 주변 상호작용, 아이템 드롭, 특수 효과 보유

#### 콤보

- 공이 바닥에 닿지 않은 상태에서 연속 파괴 시 콤보 증가
- 콤보가 오를수록 점수 배율과 연출 강도 증가
- 공이 바닥에 닿거나 생명 손실이 발생하면 콤보 초기화
- 콤보 연출 강도는 `기본 -> 강조 -> 피크` 3단계로 구분한다.
- `3콤보 이상`은 강조 단계, `8콤보 이상`은 피크 단계로 전환한다.
- 피크 단계에서는 일반 충돌 연출보다 콤보 유지와 연쇄 파괴 정보가 우선 노출된다.

#### 파워업

- 아이템 벽돌 또는 특정 보상 선택으로 획득
- 런 중 일시 효과 또는 현재 스테이지 효과 중심

### Controls and Input

#### 기본 입력

- `탭`: 게임 시작, 공 발사, 일부 액티브 아이템 사용, UI 확인
- `홀드/드래그`: 패들 이동

#### 입력 규칙

- 초기 발사 전 공은 패들 상단에 부착된 상태
- 발사 후에는 패들만 조작 가능
- 액티브 아이템이 있을 때 탭 우선순위는 “아이템 사용 가능 상태” 여부로 분기

#### 입력 감도 명세

- 패들의 화면 이동은 손가락 이동 거리와 1:1 기준으로 시작한다.
- 감도 옵션은 `0.85 / 1.0 / 1.15` 세 단계만 제공한다.
- 낮은 프레임 상황에서도 입력 샘플링이 끊기지 않아야 함
- 입력 지연 목표: 체감상 즉시 반응, 내부 기준 1프레임 이내 반영 지향
- 터치 시작 허용 영역은 화면 하단 35% 구간으로 제한한다.
- 손가락이 화면 밖으로 벗어나면 패들은 즉시 정지하며 관성은 적용하지 않는다.

## Roguelite Structure

### Run Structure

- 한 런은 기본 4개 스테이지로 구성한다.
- 목표 플레이 시간은 2분 30초에서 4분 사이로 고정한다.
- 런 시작 시 기본 패들, 기본 공 1개, 생명 3으로 시작한다.
- 스테이지 클리어 사이에는 1회만 보상 선택 또는 강화 선택을 제공한다.
- MVP 범위에서는 보스 스테이지를 제외하고 4개 일반 스테이지만 검증한다.

### Permadeath and Progression

- 런 도중 패배 시 해당 런의 일시 파워업은 소멸
- 메타 진행은 MVP에서 1개 축만 도입한다.
- MVP 메타 진행 축은 “시작 보상 선택지 확장”으로 고정한다.
- 영구 능력치 증가형 메타 진행은 MVP 범위에서 제외한다.

### Item and Upgrade System

- 아이템 타입: 즉시 효과형, 스테이지 지속형, 런 지속형
- 희귀도 시스템은 MVP에 미도입
- 빌드 다양성은 아이템 수보다 상호작용 차이로 확보

## Brick and Power-up Design

### Brick Types

#### 일반 벽돌

- 체력 1
- 기본 점수 제공
- 튜토리얼 및 초반 스테이지의 중심 타입

#### 강철 벽돌

- 체력 3
- 타격마다 균열 단계 표시
- 콤보 유지와 반사각 제어 숙련을 요구

#### 폭발 벽돌

- 파괴 시 인접 8칸 또는 정의된 범위에 피해
- 연쇄 파괴와 콤보 상승의 핵심

#### 아이템 벽돌

- 파괴 시 35% 확률로 아이템 드롭
- 보상 기대감을 만드는 장치

### Power-up Pool

#### 패들 확장

- 지속시간 12초
- 패들 길이 1.5배
- 안정성 증가용
- 중첩 시 크기 배수는 유지하고 지속시간만 최대 20초까지 연장한다.

#### 파이어볼

- 지속시간 10초
- 충돌 지점 주변 벽돌에 추가 피해
- 폭발 벽돌과 함께 광역 파괴 조합을 형성
- 중첩 시 범위는 유지하고 지속시간만 최대 18초까지 연장한다.

#### 멀티볼

- 현재 공을 3개로 분기
- 고득점과 위기 관리를 동시에 유발
- 활성 공 개수 상한은 5개로 제한한다.
- 상한 초과 시 새 공 생성 대신 기존 보조 공 유지시간만 짧게 연장한다.

#### 자석

- 지속시간 15초
- 떨어지는 아이템을 자동 흡수
- 선택 피로를 줄이고 리텐션에 유리한 편의성 파워업
- 자석 효과는 다른 파워업과 병행 가능하지만 중첩되지 않는다.

### Future Candidates

- 레이저 샷
- 패들 방패
- 슬로우 모션
- 점착 패들

## UX and Interface

### HUD Information Hierarchy

상단에 다음 정보를 배치한다.

- 현재 스테이지
- 점수
- 콤보 게이지 또는 콤보 숫자
- 생명 수

원칙:

- HUD는 한눈에 읽혀야 하며 플레이 영역을 침범하지 않는다.
- 텍스트는 크게, 정보는 적게 유지한다.

### Play Area

- 중앙은 벽돌, 공, 이펙트만 집중해서 보여주는 영역
- 배경은 과밀하지 않게 유지
- 공의 움직임과 충돌 지점이 항상 잘 보여야 함

### Readability and Main Ball Rules

- 플레이어가 동시에 추적해야 할 우선순위는 `메인 공 -> 즉시 충돌 지점 -> 떨어지는 위험/보상 요소` 순서로 고정한다.
- 멀티볼 상태에서도 메인 공 1개를 항상 지정하며, 메인 공은 나머지 공보다 밝기, 외곽선, 잔상 중 최소 1개 요소로 구분한다.
- 폭발, 파이어볼, 콤보 이펙트는 메인 공 외곽을 완전히 덮지 않아야 하며, 메인 공 실루엣이 0.2초 이상 사라지면 안 된다.
- 아이템 드롭과 파워업 획득 연출은 메인 공 추적보다 우선하지 않는다.
- 실패 직전 마지막 공은 자동으로 메인 공으로 승격되며, 시선 유도 효과를 한 단계 강화한다.

### Bottom Area

- 패들 조작 중심 구역
- UI 버튼 최소화
- 손가락이 가리는 위치를 고려한 안전영역 확보

### Onboarding

- 첫 진입 시 1스크린 이하의 초간단 튜토리얼
- “드래그로 이동, 탭으로 시작”을 즉시 체험시키는 구조
- 튜토리얼 텍스트보다 유도 애니메이션 우선

### Failure and Retry UX

- 실패 순간 짧은 슬로우/정지 후 결과 표시
- 5초 이내 재도전 가능
- “한 판 더” 버튼을 가장 강조
- 실패 연출은 0.6초 이내, 결과 UI 노출은 1.5초 이내에 완료한다.
- 결과 UI 상태는 `실패 연출 -> 결과 표시 -> 재도전 가능 -> 런 종료 선택 가능` 순으로 고정한다.
- 재도전 선택 시 추가 확인 없이 즉시 새 런의 첫 스테이지로 진입한다.

### Feedback Intensity Rules

- 피드백 강도는 `기본`, `강조`, `피크` 세 단계로 관리한다.
- 기본 단계: 일반 벽돌 파괴, 일반 반사, 일반 점수 획득. 짧은 파티클과 짧은 충돌음만 사용한다.
- 강조 단계: 강철 벽돌 파괴 직전, 폭발 벽돌 연쇄 시작, 3콤보 이상, 보상 선택 진입. 화면 흔들림과 사운드 레이어를 1단계 올린다.
- 피크 단계: 8콤보 이상, 스테이지 클리어, 마지막 생명 위기, 대형 연쇄 폭발. 히트 스톱, 카메라 반응, 강조 사운드를 모두 사용하되 0.35초 이상 시야를 방해하지 않는다.
- 같은 프레임 또는 같은 0.2초 윈도우 안에서 여러 피크 이벤트가 겹치면 가장 중요한 1개만 피크로 처리하고 나머지는 강조 단계로 낮춘다.
- 중요도 우선순위는 `실패/생존 위기 -> 메인 공 충돌 -> 스테이지 클리어 -> 콤보 피크 -> 아이템 획득` 순으로 고정한다.

### Accessibility

- 충분한 텍스트 크기
- 색약 대응 가능한 벽돌 상태 표시
- 진동/화면 흔들림 강도 옵션 제공 가능성 검토

## Progression and Balance

### Player Progression

- 런 내 진행: 스테이지 클리어 -> 보상 선택 -> 난이도 상승
- 메타 진행: MVP에서는 1개 축만 유지
- MVP 확정안: 시작 보상 선택지 확장
- 4개 스테이지 기준 한 런에서 보상 선택은 총 3회 발생한다.

### Difficulty Curve

- 초반 30초: 성공 경험 제공
- 1분 내: 강철 벽돌과 콤보 유지 압박 노출
- 2분 이후: 폭발 벽돌, 멀티볼 등 변수가 늘어나며 판단 요구
- 후기: 공 속도 증가, 배치 복합화, 특수 벽돌 비율 증가

### Emotional Curve by Stage

- 스테이지 1: “쉽고 시원하다”가 핵심 감정이다. 첫 성공, 첫 연쇄 파괴, 첫 강한 타격감을 빠르게 제공한다.
- 스테이지 2: “내가 컨트롤하고 있다”는 감정을 만든다. 각도 제어와 강철 벽돌 대응으로 숙련감을 전달한다.
- 스테이지 3: “조합이 터진다”는 감정을 만든다. 폭발 벽돌, 파워업, 콤보 강조 구간이 자연스럽게 이어져야 한다.
- 스테이지 4: “혼란을 이겨낸다”는 감정을 만든다. 멀티볼과 복합 배치가 나오지만 판독성과 통제감은 유지되어야 한다.
- 각 스테이지는 이전 스테이지보다 단순히 어려워지는 것이 아니라, 새로운 감정 피크를 하나씩 제공해야 한다.

### Economy and Resources

- 기본 자원: 점수, 생명, 콤보
- 선택 자원: 스테이지 보상 선택지
- MVP에서 소프트/하드 화폐는 도입하지 않음
- 점수는 메타 화폐로 직접 변환되지 않는다.
- 보상 선택지는 항상 2개 중 1개 선택으로 제한한다.

## Level Design Framework

### Level Types

1. 튜토리얼형: 일반 벽돌 중심
2. 리듬형: 일정 패턴과 연쇄 파괴 유도
3. 제어형: 강철 벽돌과 좁은 각도 플레이 비중 증가
4. 카오스형: 폭발 벽돌, 아이템 벽돌, 멀티볼 시너지 강조
5. 특수형: 보스 또는 기믹형 스테이지

### Level Progression

- 스테이지는 짧고 명확해야 함
- 한 스테이지의 목적은 하나의 학습 또는 하나의 쾌감 포인트에 집중
- 초반 4스테이지는 MVP 핵심 검증 구간으로 고정한다.
- 스테이지 1은 성공 경험, 2는 각도 제어, 3은 조합 반응, 4는 카오스 통제 학습에 집중한다.
- 각 스테이지의 첫 15초 안에는 그 스테이지의 핵심 감정을 예고하는 장면이 반드시 1회 이상 발생해야 한다.
- 각 스테이지 종료 직전 10초에는 클리어 가능성이 높아지는 마무리 리듬을 제공해, 좌절보다 다음 스테이지 기대를 남긴다.

### Boss / Special Stage Direction

- MVP 필수는 아님
- 단, GDD 단계에서는 확장 슬롯으로 정의
- 보스안 예시: 이동형 핵심 벽돌, 보호막 패턴, 약점 노출 타이밍

## Art and Audio Direction

### Art Style

- 밝고 선명한 아케이드 컬러
- 벽돌 타입별 실루엣과 상태 차이가 분명해야 함
- 이펙트는 큰 반응을 우선하고 배경은 정리된 톤으로 유지

### Audio and Music

- 공 충돌, 벽돌 파괴, 콤보 상승에 명확한 사운드 계층 제공
- 짧은 루프형 BGM 사용
- 보상 선택, 클리어, 실패 순간에 청각적 강조 배치
- 메인 공 충돌음은 다른 효과음보다 항상 명확히 들려야 하며, 멀티볼 상태에서도 존재감이 유지되어야 한다.
- 콤보 단계 상승, 위기 상태, 스테이지 클리어는 서로 다른 청각 서명을 가져야 한다.

## Technical Specifications

### Performance Requirements

- 목표 프레임: 60fps
- 낮은 성능 기기에서도 입력 누락 없이 플레이 가능해야 함
- 이펙트는 과감하게 보이되 성능 예산 안에서 단계별 품질 조절 가능해야 함
- 성능 저하 시 축소 대상은 배경/잔상/보조 파티클 순이며, 메인 공 강조 효과와 핵심 충돌 피드백은 마지막까지 유지한다.

### Platform-Specific Details

- HTML5 Canvas + JavaScript 기반
- 모바일 브라우저 최적화 우선
- 해상도 기준은 1080 x 1920, 실제 렌더링은 반응형 스케일
- 세로 고정 플레이 기준

### System Modules

1. `InputSystem`
   - 터치 시작/이동/종료 처리
   - 상대 드래그 계산
2. `GameLoop`
   - update/render 분리
   - 상태 전이 관리
3. `PhysicsSystem`
   - 공 이동, 반사, 충돌 판정
4. `BrickSystem`
   - 체력, 파괴, 특수 효과
5. `PowerupSystem`
   - 드롭, 획득, 효과 지속
6. `UISystem`
   - HUD, 튜토리얼, 실패/클리어, 보상 선택
7. `SaveState`
   - 짧은 세션 복구 또는 기본 진행 상태 저장

### Asset Requirements

- 패들 1종
- 공 1종
- 벽돌 최소 4종 비주얼
- 파워업 아이콘 최소 4종
- 히트/파괴 파티클 세트
- HUD 아이콘 및 숫자 폰트 리소스

## 2026-03-29 Senior Design Review Amendments

본 개정 섹션은 `Samus Shepard` 중심 시니어 디자인 리뷰와 Party Mode 논의를 반영한다. 목적은 현재 빌드가 "기능 검증용 테스트 게임"처럼 보이는 원인을 제거하고, 작은 범위 안에서도 상용 아케이드 게임 수준의 타격감과 시청각 보상 밀도를 확보하는 것이다.

### Review Summary

- 현재 코어 루프와 조작 구조는 충분히 유효하다.
- 문제는 시스템 부족보다 "순간 체감 품질" 부족에 있다.
- 브릭, 패들, 공, HUD, 보상 카드가 모두 기능 표시물처럼 보여 상용 감각을 약화시킨다.
- 충돌 순간의 히트스톱, 파편, 발광, 사운드 레이어, 배경 긴장도 변화가 얕다.
- Stage 1~4가 규칙적으로는 구분되지만 감정적으로는 충분히 다르게 느껴지지 않는다.

### Commercial Feel Pillars

#### 1. Hit Feel Must Be Layered

모든 충돌은 단순 반사로 끝나지 않고 최소 3층의 피드백을 제공해야 한다.

- 물리층: 반사각, 속도 유지, 패들 중심 대비 타점 차이
- 시각층: 스파크, 짧은 섬광, 링 파동, 꼬리 강화, 균열 단계
- 청각층: 브릭 타입별 명확히 다른 타격음, 콤보 단계별 누적 사운드

일반 브릭은 빠르고 가벼운 쾌감, 강체 브릭은 묵직한 저항감, 폭발 브릭은 순간적인 쇼크와 연쇄 기대감, 아이템 브릭은 보상 예고감을 줘야 한다.

#### 2. Readability Over Raw Spectacle

화려함은 허용되지만 언제나 메인 공, 충돌 지점, 위험/보상 대상의 판독성이 우선이다.

- 멀티볼 상황에서도 메인 공은 가장 밝은 코어와 가장 긴 꼬리를 가진다.
- 폭발 및 레이저 연출은 0.3초 이내에 주요 결과를 읽을 수 있어야 한다.
- 피크 연출 중첩 시 가장 중요한 하나만 풀 강도로 출력하고 나머지는 하향한다.

#### 3. Commercial Reward Cadence

보상은 "기능 설명"이 아니라 "선택하고 싶은 상품"처럼 보여야 한다.

- 카드에는 짧은 헤드라인, 아이콘, 한 줄 효과 설명, 즉시 체감 포인트가 들어간다.
- 보상 선택 직후에는 화면 전환이 아니라 짧은 확인 연출과 다음 런 기대감을 준다.
- 아이템 선택의 감정은 정보 전달보다 "강화됐다"는 기분을 우선해야 한다.

### Material and Visual Identity Rules

현재의 단색 도형 스타일은 개발 편의상 유지할 수 있으나, 연출 레벨에서는 재질감과 계층을 강하게 드러내야 한다.

- 일반 브릭: 밝은 에너지 코어와 깨지는 가장자리
- 강체 브릭: 금속/합금 계열, 균열 누적, 타격 시 무거운 잔광
- 폭발 브릭: 고온 코어, 위험 경계선, 터지기 직전 점멸
- 아이템 브릭: 내부 보상 심볼과 수집 기대를 주는 펄스
- 패들: 단순 바가 아니라 장비/모듈 느낌의 3단 구조 실루엣
- 공: 중심 코어, 외곽 halo, 속도에 따라 꼬리 길이 변화

색상만으로 상태를 설명하는 것은 금지한다. 브릭 타입, 파워업, 콤보 단계는 반드시 실루엣, 균열, 아이콘, 발광 리듬 중 최소 두 가지 이상으로 구분되어야 한다.

### Power-up Spectacle Rules

각 파워업은 기능뿐 아니라 즉시 체감되는 연출 서명을 가져야 한다.

- Expand: 패들이 순간적으로 옆으로 펼쳐지는 전개 애니메이션과 금속 잠금 해제 사운드
- Laser: 발사 전 짧은 레인 예고선, 발사 후 잔광과 충돌 지점의 열 흔적
- Multiball: 공이 복제되는 것이 아니라 분열되는 느낌의 split flash와 분기 궤적
- Magnet: 패들 상단 자력 필드 표시, 흡착 시 snap-back 반응과 짧은 자성음

파워업은 단순 수치 상승처럼 느껴져서는 안 되며, 플레이 스타일이 바뀌는 순간을 분명하게 연출해야 한다.

### Stage Emotion Curve Revisions

각 스테이지는 난이도 증가가 아니라 다른 감정 피크를 제공해야 한다.

- Stage 1: "즉시 시원함"
  - 첫 20초 안에 파괴 쾌감을 느끼게 하고 정보 부담을 낮춘다.
- Stage 2: "정밀 제어"
  - 강체 브릭과 레이저/자석 선택으로 제어의 재미를 강조한다.
- Stage 3: "연쇄 폭발"
  - 폭발 브릭, 콤보, 화면 반응을 통해 쇼타임 감각을 만든다.
- Stage 4: "혼돈 속 통제"
  - 멀티볼과 피크 연출이 겹치되, 판독성 정책으로 통제 가능한 카오스를 유지한다.

### Failure and Retry Drama

실패는 허무하게 끝나면 안 되며, 짧더라도 감정적 punctuation을 가져야 한다.

- 마지막 공 손실 시 0.15~0.25초 슬로우 또는 타격 정지
- 화면 전체를 덮는 암전 대신 중심부 수축형 실패 반응 사용
- 결과/재도전 UI는 즉시 접근 가능하되, 실패 원인을 한눈에 읽을 수 있어야 한다
- "다시 하기"는 가장 시각적으로 강한 CTA로 유지한다

### Anti-Prototype Guardrails

다음 항목은 상용 품질 목표와 충돌하므로 금지한다.

- 텍스트만 있는 보상 카드
- 평면 사각형만으로 끝나는 파워업 시각 표현
- 브릭 타입을 색상만으로 구분하는 처리
- 콤보 피크에도 변화가 없는 HUD
- 실패/클리어 순간에 게임 화면과 감정 온도가 거의 같게 유지되는 구성

### Required Follow-up for Stories and Implementation

- 모든 브릭 타입에 재질 차이와 피격 반응 단계를 정의할 것
- `feedback/presentation` 레이어에서 브릭/패들/공의 연출 토큰을 데이터화할 것
- 보상 카드 UI를 "상품 카드" 수준으로 재설계할 것
- Stage 1~4별 배경, 타이포 강조, 사운드 텐션, HUD accent 변화를 명시할 것
- 콤보 피크, 폭발 연쇄, 멀티볼 분열, 레이저 발사를 별도 하이라이트 이벤트로 취급할 것
